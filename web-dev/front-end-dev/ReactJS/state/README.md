This folder contains examples and patterns for managing `state in React`. Helpful reference while building real applications.

- In React, `state` represents any data that changes over time and causes the UI to re-render.
- The examples here will show `how state behsaves`, `how it flows`, and `how it interacts with user actions and async logic`.

---
## What This Folder Covers

Topics in this folder include:
- Primitive state (`useState` basics)
- Async state lifecycles (loading, error, success)
- Derived state (state computed from other state)
- Conditional UI rendering driven by state
- Form state and validation
- Object and array state updates (immutability)
- State lifecycle patterns (resetting, syncing, cleanup)
- Local vs global state concepts (preparing for Redux/Zustand)
